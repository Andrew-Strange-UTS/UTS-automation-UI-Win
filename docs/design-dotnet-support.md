# C# / .NET support

**Coming soon.**

Today Marvin tests are written in JavaScript (`run.js`). Support for writing
tests in **C# / .NET** is planned, so testers and developers can author
automation in a language they already use.

What is planned:

- Author a test as `run.cs` and have it appear as a test card, the same way a
  `run.js` test does.
- The same capabilities C# tests would have as JS tests: window control,
  keyboard and mouse, screenshots, OCR and image matching for desktop, and
  Selenium for web.
- Parameters and secrets available to the test, and per-step Zephyr reporting.
- Mixing JS and C# steps in a single run sequence.

There is no date yet. This page will be updated when the work is scheduled.
